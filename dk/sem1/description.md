## 

Необходимо разработать архитектуру приложения путеводителя.

### исходные файлы:

- диаграмма прецедентов для приложения находится в файле ./diagram.pdf
- все диаграммы в графическом представлении находятся в файле ./all-diagrams.pdf

приложение должно состоять их бекенда и фронтеда

### технологии на бекенде:

- java
- spring-boot
- maven
- jpa
- hibernate
- postgres 
- swagger

бекенд должен быть реализован в виде REST API

### Технологии на фронтеде

- typescript
- react
- react-router
- html
- css

### задача 1

Разработать архитектуру бекенда согласно требованиям из диаграммы прецедентов. добавить возможность аутентификации пользователя по логину и паролю, после логина пользователь должен получать токен для авторизации. Начни с анализа диаграм и предложи структуру базы данных для этого проекта. 

Сохрани описание структуры базы данных в файл ./database.md
