# Структура базы данных

## 1. Таблица `users` (Пользователи)
Для хранения информации об учетных записях пользователей и аутентификации.

| Поле            | Тип данных     | Описание                              |
| :-------------- | :------------- | :------------------------------------ |
| `id`            | `BIGINT` (PK)  | Уникальный идентификатор пользователя |
| `username`      | `VARCHAR(255)` | Уникальное имя пользователя (логин)   |
| `password_hash` | `VARCHAR(255)` | Хэш пароля пользователя               |
| `token`         | `VARCHAR(255)` | Токен для авторизации                 |



## 2. Таблица `user_form` (Анкеты)

Для хранения данных анкеты, заполняемой пользователем (UC-01). Связана с таблицей `users`.

| Поле                               | Тип данных         | Описание                                                              |
| :--------------------------------- | :----------------- | :-------------------------------------------------------------------- |
| `id`                               | `BIGINT` (PK)      | Уникальный идентификатор анкеты                                       |
| `user_id`                          | `BIGINT` (FK)      | Ссылка на `users.id`                                                  |
| `entry_date`                       | `DATE`             | Дата въезда                                                           |
| `country_id`                       | `BIGINT` (FK)      | Ссылка на `countries.id` (Страна гражданства)                         |
| `visit_purpose`                    | `VARCHAR(255)`     | Цель визита                                                           |
| `resettlement_program_status`      | `CHAR(1)`    | Статус по программе переселения ("да" - Y, "нет" - N, "член семьи" - F) |
| `highly_qualified_specialist_status`| `CHAR(1)`    | Статус по высококвалифицированному специалисту  ("да" - Y, "нет" - N, "член семьи" - F) |



## 3. Таблица `roadmap_steps` (Шаги дорожной карты)

Справочник шагов дорожной карты

| Поле          | Тип данных    | Описание                      |
| :------------ | :------------ | :---------------------------- |
| `id`          | `BIGINT` (PK) | Уникальный идентификатор шага |
| `step_number` | `INT`         | Порядковый номер шага         |
| `description` | `TEXT`        | Описание шага                 |



## 4. Таблица `user_steps` (Шаги дорожной карты)

Для хранения сгенерированных шагов дорожной карты для каждого пользователя (UC-02) и отслеживания их статуса.

| Поле      | Тип данных        | Описание                                       |
| :-------- | :---------------- | :--------------------------------------------- |
| `step_id` | `BIGINT` (PK, FK) | Уникальный идентификатор шага                  |
| `user_id` | `BIGINT` (PK, FK) | Ссылка на `users.id`                           |
| `status`  | `VARCHAR(50)`     | Статус шага (например, "pending", "completed") |



## 5. Таблица `countries` (Страны)

Для хранения справочника стран и использования в качестве внешнего ключа в `user_profiles`.

| Поле   | Тип данных     | Описание                                    |
| :----- | :------------- | :------------------------------------------ |
| `id`   | `BIGINT` (PK)  | Уникальный идентификатор страны             |
| `name` | `VARCHAR(255)` | Название страны (например, "Россия", "США") |
